Object Oriented Programming (OOP) merupakan pendekatan desain sistem yang meniru cara manusia memahami dunia nyata dengan merepresentasikan entitas sebagai objek yang memiliki karakteristik dan perilaku, sehingga program menjadi lebih intuitif dan terstruktur; OOP meningkatkan modularitas melalui pembagian program ke dalam class yang terorganisir, mendukung reusability agar kode dapat digunakan kembali, serta memiliki empat pilar utama yaitu Encapsulation, Abstraction, Inheritance, dan Polymorphism yang membuat sistem lebih fleksibel, aman, scalable, maintainable, dan robust, serta menjadi fondasi penting dalam pengembangan aplikasi modern seperti Flutter berbasis Dart.

1. Class & Object (Car)

Dalam konteks OOP, class berfungsi sebagai representasi abstrak dari suatu entitas di dunia nyata. Class tidak hanya menyimpan data, tetapi juga mendefinisikan bagaimana data tersebut diproses melalui method. Dengan kata lain, class menggabungkan data dan fungsi dalam satu kesatuan yang utuh.

Sedangkan object merupakan implementasi nyata dari class. Ketika sebuah object dibuat, memori dialokasikan untuk menyimpan nilai attribute yang dimilikinya. Setiap object memiliki state yang bisa berbeda meskipun berasal dari class yang sama. Hal ini memungkinkan fleksibilitas dalam pemodelan sistem.

Pada contoh class Car, attribute seperti model dan color merepresentasikan karakteristik mobil, sementara method start() dan stop() menggambarkan perilaku dasar mobil tersebut. Pendekatan ini membuat struktur program lebih modular dan mudah dikembangkan. 

    class Car {
  String? model;
  String? color;

  void start() {
    print('Car started!');
  }

  void stop() {
    print('Car stopped!');
  }
}

void main() {
  Car myCar = Car();
  myCar.model = "Sedan";
  myCar.color = "Merah";

  myCar.start();
  myCar.stop();
}

2. Constructor

Constructor adalah metode inisialisasi yang dipanggil secara otomatis saat objek baru diciptakan dalam memori sistem. Di Dart, kita bisa menggunakan formal parameters (this.attribute) untuk menyederhanakan proses pengisian nilai attribute. Constructor merupakan metode khusus yang dipanggil secara otomatis saat objek dibuat. Constructor berfungsi untuk melakukan inisialisasi nilai awal attribute sehingga objek langsung berada dalam kondisi valid dan siap digunakan. Dalam bahasa Dart, constructor dapat menggunakan formal parameter (this.attribute) untuk menyederhanakan penulisan kode.

3. Validasi Constructor (Mahasiswa)

Validasi pada constructor merupakan teknik untuk memastikan bahwa data yang diberikan saat pembuatan object berada dalam kondisi yang benar (valid). Tanpa validasi, object berpotensi memiliki nilai yang tidak logis atau berada di luar batas yang diperbolehkan.

Dalam contoh class Mahasiswa, validasi dilakukan terhadap nilai IPK agar tetap berada dalam rentang 0.0 hingga 4.0. Jika nilai yang dimasukkan berada di luar batas tersebut, sistem dapat memberikan peringatan atau menolak pembuatan object. Hal ini bertujuan menjaga konsistensi dan integritas data akademik.

Dengan menerapkan validasi pada constructor, kita memastikan bahwa setiap object yang dibuat sudah memenuhi aturan bisnis (business rule) sejak awal. Pendekatan ini mengurangi kemungkinan terjadinya error pada proses eksekusi program di tahap berikutnya.

4. Encapsulation (Person)

Encapsulation adalah teknik untuk menyembunyikan detail implementasi internal. Di Dart, kita menggunakan awalan underscore (_) untuk membuat attribute bersifat private, sehingga hanya bisa diakses melalui getter dan setter. prinsip fundamental dalam OOP yang bertujuan melindungi data internal objek dari akses langsung yang tidak sah. Dengan membatasi akses melalui mekanisme getter dan setter, integritas data dapat dijaga dan kesalahan manipulasi dapat diminimalisir. Encapsulation meningkatkan keamanan serta stabilitas sistem dalam skala besar.

5. Named Parameter (Produk)

Latihan lanjutan menggunakan Named Parameters dan operator required. Teknik ini membuat pembuatan objek menjadi lebih deskriptif dan aman karena compiler akan memberikan peringatan jika data wajib tidak diisi. Selain itu, Dart juga mendukung penggunaan Named Parameter, yaitu teknik pengisian parameter berdasarkan nama variabel. Penggunaan keyword required memastikan bahwa parameter penting tidak boleh kosong saat proses instansiasi objek dilakukan. Pendekatan ini meningkatkan kejelasan kode (readability) dan mengurangi potensi kesalahan input.

Kesimpulan

Secara keseluruhan, Object Oriented Programming (OOP) merupakan paradigma yang efektif untuk membangun sistem yang terstruktur dan modular. Melalui konsep class, object, constructor, validasi, dan encapsulation, program menjadi lebih aman, jelas, mudah dikembangkan, serta lebih terjaga dalam jangka panjang, terutama dalam pengembangan aplikasi menggunakan Dart dan Flutter.
